# Security Policy

## Privacy-first rule

This repository is designed to be safe to publish. The bundled examples are synthetic and should stay synthetic.

Do not commit:

- API keys or access tokens
- `.env` files or local secrets
- customer data or personal data
- private task histories
- internal incident reports
- local absolute paths
- production logs
- private screenshots or exports

## Reporting a vulnerability

If you find a security issue, open a GitHub issue with a minimal description and avoid posting live credentials or private data.

## Recommended use

Run the workbench locally first. If you adapt it for an organization, keep real operational data in a private repository or an internal system.
