# Audit Checklist

## Safety

- [ ] No credentials or tokens.
- [ ] No private user data.
- [ ] No local absolute paths.
- [ ] No production logs.
- [ ] No unreviewed public release.

## Quality

- [ ] README explains the purpose in 30 seconds.
- [ ] Examples are synthetic.
- [ ] Templates are reusable.
- [ ] The app runs locally.
- [ ] License is present.

## Evidence

- [ ] Directory tree captured.
- [ ] Secret scan captured.
- [ ] Privacy scan captured.
- [ ] Review verdict captured.
